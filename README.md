# drive-picture-scores

## 概要

懇親会の写真を AI が自動採点し、大画面で得点演出と暫定ランキングを出すイベント用 Web アプリ。

## 設計

フロントは React + Vite + framer-motion の SPA。ルータは使わず `pathname` ベースで「採点 / 結果発表 / 設定」の 3 画面を切り替える。バックエンドは Node/Express。Google Drive の共有フォルダから画像一覧を取得し、新しい写真を見つけるたびに採点してキューに積む。採点は Claude Vision（structured outputs）で画素を読み、4 軸スコアと人数などのシグナルを返す。API キー未設定時はダミー採点へ自動フォールバックする。結果は Google スプレッドシートを簡易 DB として保存し、書き込みは Apps Script の Web App（課金・サービスアカウント不要）を既定経路に、画像 ID をキーに upsert する。結果発表は公開シートの CSV を読み、スキャン済みの上位 3 件を集計。Drive フォルダ ID・シート ID・Webhook URL・API キーなどの秘匿値は `.env`（gitignore）と `/settings`（サーバー永続）で管理し、リポジトリには載せない。

## 仕様

採点軸は「笑顔・表情 / にぎやかさ(人数) / 構図・遠近 / 写りの良さ」の 4 つ。配点は人数を主役に 20/40/20/20、各軸は最低 10 点からの加点方式で総合 10〜100 点に収まる。にぎやかさは実際の頭数だけで決まり、人数が多いほど高得点（多人数で頭打ち）、画像ごとに安定した ±3 点の遊びを加える。容姿・年齢・性別は一切評価しない。採点画面では写真を実アスペクト比で全体表示（縦横とも見切れない）、得点はカウントアップ演出後に内訳パネルが右からせり出す。司会操作として右下「次へ」・左下「戻る」で手動送りができる。結果発表は 1 位を左半分に大きく、2・3 位を右半分の上下に置き、順位と得点を写真の上にオーバーレイ表示する。設定画面ではドライブ URL と保存先 Web App URL を編集でき、ランキングのリセット（確認ダイアログで OK / いいえ → シートをヘッダのみに戻し集計を初期化 → 1 から再スキャン）も実行できる。サーバー再起動後もシートのスキャン済み ID を読み込み、同じ写真の再採点を防ぐ。内訳パネルには 4 軸の配点と合計に加え、にぎやかさの後ろへ実スキャン人数（取得できなければ No entry）を小さく表示する。総合点は S / A / B / C / D のグレードと連動し、リビール時は点数とリアクションコメントを大きく見せる。画像の取得元は公開 Drive フォルダ・サービスアカウント・サンプルのモックの 3 方式から選べる。

> 採点の詳細な配点・計算式は [README-SCORING.md](./README-SCORING.md) を参照。
