# 判定評価

懇親会で参加者みんなに撮ってもらった写真を、カラオケの採点のように
点数化して大画面で発表するための「判定軸」と「採点ルール」の定義です。

## メインテーマ

この判定で重要視するのは以下の **四つの観点** です。

- **人数が多いか／大きく写っているか** … みんなで撮った“にぎやかさ”
- **笑顔が多いか** … 表情・場の楽しさ
- **遠近法がとてもいいか** … 構図・奥行き・写真としての気持ちよさ
- **顔がわかりやすく写っているか／描写が綺麗か** … 誰が写っているか伝わる写りの良さ

ひとことで言うと「**みんなが・楽しそうに・気持ちよく・はっきり**写っている写真」を高く評価します。

---

## 大前提：投稿者が傷つかない採点にする

この採点は“盛り上げるためのお遊び”です。誰かをイジったり、容姿を格付けする道具には絶対にしません。
そのために次のルールを守ります。

1. **容姿（美人・かっこいい等）は評価軸にしない。**
   「美人画像」「おじさんだけ」などの“被写体の見た目そのもの”は点数に一切影響させません。
   評価するのは *写真の撮り方・場の楽しさ* であって、*写っている人の顔の優劣* ではありません。
2. **減点方式ではなく加点方式。** 「できていない」を引くのではなく「できている」を足します。
3. **点数に“下駄（最低点）”を履かせる。** どの写真も一定点からスタートし、極端な低得点が出ないようにします
   （後述の `FLOOR`）。最下位でも「惜しい！」で済むレンジに収めます。
4. **評価は“写真単位”であって“個人単位”ではない。** 「この人が低い」という見せ方は一切しません。
5. **グレード表記は全部ポジティブ。** D評価でも「ナイスショット」など前向きな言葉にします（後述）。
6. **コメントは必ず良かった点から。** 自動生成コメントは長所を先に、改善は“次はこうするともっと”の口調で。

---



## 判定軸の定義（4軸）

各軸は内部的に **0〜100** で採点し、`FLOOR`（既定 30）からスタートする“加点”として扱います。
表示上は **各軸を均等配分（100 ÷ 4 軸 ＝ 各 25 点満点）** に直し、4軸の配点を合計したものが総合スコアになります。

### 1. にぎやかさ（人数・占有感） `people`
「何人で・どれだけ大きく」写っているか。みんなで撮った写真をいちばん盛り上げたいので主軸の一つ。

| 見るポイント | 高得点 | 低得点 |
| --- | --- | --- |
| 写っている人数 | たくさん | ひとり／無人 |
| 顔の大きさ（画面占有率） | 大きくしっかり | 小さく豆粒 |
| 全員が画角に収まっているか | きれいに収まる | 見切れが多い |

> 「大きく写っているか」は **顔の面積の合計が画面に占める割合** で測ります（顔が大きい＝距離が近い＝仲が良さそう、という発想）。

### 2. 笑顔・表情 `mood`
場の楽しさ。笑顔の“数”と“強さ”を見ます。にぎやかさと並ぶ最重視軸。

| 見るポイント | 高得点 | 低得点 |
| --- | --- | --- |
| 笑顔の人数の割合 | 多くの人が笑顔 | 無表情が多い |
| 笑顔の強さ | はじける笑顔 | 控えめ |
| 場の一体感（みんな同じノリか） | そろっている | バラバラ |

> 真顔やクールショットを“悪い”とはしません。あくまで笑顔が多いほど**加点が伸びる**だけです。

### 3. 構図・遠近感 `composition`
「遠近法がとてもいいか」。写真としての気持ちよさ・奥行き・バランス。

| 見るポイント | 高得点 | 低得点 |
| --- | --- | --- |
| 奥行き・遠近感 | 立体的で引き込まれる | のっぺり |
| 構図バランス（日の丸／三分割など） | 安定 or 効果的 | 偏り過ぎ |
| 水平・傾き | 整っている | 不自然に傾く |
| 背景の主張 | 主役が引き立つ | ごちゃつき |

### 4. 写りの良さ（顔のわかりやすさ・描写） `clarity`
「誰が写っているか伝わるか」「描写が綺麗か」。ピント・明るさ・ブレ。

| 見るポイント | 高得点 | 低得点 |
| --- | --- | --- |
| ピント | 顔にしっかり合う | ボケている |
| 明るさ・露出 | 顔が見える明るさ | 暗い／白飛び |
| ブレ | 止まっている | 手ブレ |
| 顔の向き | カメラを向いている | 後ろ向きばかり |

## 重み付け（ウェイト）＝ 均等配分

4軸を **均等** に扱います。各軸の持ち点は **100 ÷ 4 ＝ 25 点**、4軸の配点を足したものが総合スコアです。
（重みは全軸 `0.25`、合計 1.0。重み付き平均でも“各25点満点の合計”でも結果は同じになります。）

| 軸 | キー | 持ち点 | 重み |
| --- | --- | --- | --- |
| 笑顔・表情 | `mood` | **25点** | 0.25 |
| にぎやかさ（人数） | `people` | **25点** | 0.25 |
| 構図・遠近感 | `composition` | **25点** | 0.25 |
| 写りの良さ | `clarity` | **25点** | 0.25 |
| | 合計 | **100点** | 1.00 |

> 右下のフローティングボタン（得点が確定して表示されている瞬間だけ押せる）から、
> その一枚の「各軸 ◯/25 点」と「4軸の合計 ＝ 表示スコア」を確認できます。

## 採点の計算式

```
各軸スコア   axisValue = clamp(FLOOR + 加点, 0, 100)        // FLOOR=30 を底に加点
各軸の配点   axisPoints = axisValue / 100 × 25             // 25点満点に均等配分
総合スコア   score = Σ axisPoints                          // ＝ 4軸の合計（30〜100）
```

- `FLOOR`（最低点の下駄）：既定 **30**。各軸が必ず30以上になるので、**総合スコアは常に 30 より上**になります。
- 表示する各軸の配点は端数を整数に丸めても **合計が表示スコアにぴったり一致** するよう配分します
  （最大剰余法／ハミルトン配分）。

### グレード（全部ポジティブ表記）

| 総合点 | グレード | 表示ラベル例（傷つかない言い回し） |
| --- | --- | --- |
| 90〜100 | **S** | 伝説の一枚！🏆 |
| 80〜89 | **A** | 主役級ショット✨ |
| 65〜79 | **B** | いい雰囲気！👍 |
| 45〜64 | **C** | ナイストライ！😄 |
| 0〜44 | **D** | 味わいの一枚🎉（“ネタ枠”として笑いに変える）|

> グレードはあくまで“順位の演出”。最下位でも会場が笑顔になる言葉だけを使います。

## 実装との対応

採点ロジックは [server/scorer/index.js](server/scorer/index.js) の `scoreImage()` に集約されています
（現状はピクセルを見ないダミー実装で、本物のAI判定を後から差し替えられる形）。返り値の形：

```js
scoreImage(item, { theme, weights }) -> {
  score:     number,   // 0..100
  grade:     "S" | "A" | "B" | "C" | "D",
  breakdown: [{ key, label, value /* 0..100 */, weight }]
}
```

この4軸へ寄せるための対応表（実装の軸キー → 本ドキュメントの軸）：

| 実装の軸（現状） | 本ドキュメントの軸 | `.env` の重み変数 |
| --- | --- | --- |
| `mood`（表情・雰囲気） | 笑顔・表情 | `W_MOOD` |
| （新設）`people` | にぎやかさ（人数） | `W_PEOPLE` |
| `composition`（構図） | 構図・遠近感 | `W_COMPOSITION` |
| （`color`+`impact`を統合）`clarity` | 写りの良さ | `W_CLARITY` |
| `themeMatch`（お題一致） | ※任意。お題企画時のみ使用 | `W_THEME` |

`.env`（[.env.example](.env.example)）の推奨設定：

```dotenv
THEME=最高の乾杯
# 均等配分（各軸25点）
W_MOOD=0.25
W_PEOPLE=0.25
W_COMPOSITION=0.25
W_CLARITY=0.25
# 最低点の下駄（総合スコアは常に30より上になる）
FLOOR=30
```

### 本物のAI判定に差し替えるときの指針

各軸を、傷つけずに数値化できる**客観指標**に寄せて算出します（顔検出・表情推定などを利用）。

- `people` … 顔検出の **件数** と **顔バウンディングボックス面積の合計÷画像面積**
- `mood` … 笑顔スコア（smile/happy 確率）の **人数平均**＋笑顔人数の割合
- `composition` … 三分割交点・水平線・被写体サイズの分布から奥行き/バランスを推定
- `clarity` … ピント（ラプラシアン分散などの鮮鋭度）・明るさヒストグラム・顔向き

> AI判定でも **容姿の美醜・年齢・性別は点数に使わない**。これは本ドキュメント冒頭の大前提です。
> 推定した値はそのまま見せず、`FLOOR` を通して“加点”に変換してから表示します。
